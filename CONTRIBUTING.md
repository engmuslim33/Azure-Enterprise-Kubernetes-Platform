# Contributing

This repository is primarily a personal portfolio and training lab. Contributions should keep the project practical, repeatable, and cost-aware.

## Standards

- Prefer clear step-by-step documentation.
- Include verification commands after implementation commands.
- Add troubleshooting notes when a command may fail.
- Keep Azure Free or Student subscription limitations in mind.
- Never commit real secrets, tenant IDs, subscription IDs, passwords, or private keys.
